# 🚀 视频处理使用指南

## 📋 快速开始

### 1️⃣ 第一步：配置系统

访问设置页面：http://localhost:3000/settings

#### 配置 ASR 引擎（三选一）

**方案 A: 本地免费**
```
ASR 引擎：Faster-Whisper
ASR 模型：Medium
```

**方案 B: 云端高速**
```
ASR 引擎：Deepgram
ASR 模型：Nova-3
API Key: dg_xxxxxxxxxx
```

#### 配置 LLM 大模型（三选一）

**方案 A: 本地免费**
```
LLM 后端：本地 Ollama
LLM 模型：qwen2.5:7b
```
需要先安装 Ollama：
```bash
brew install ollama
ollama pull qwen2.5:7b
```

**方案 B: 阿里百炼**
```
LLM 后端：阿里百炼
LLM 模型：qwen3.5
API Key: sk-xxxxxxxxxx
Base URL: https://coding.dashscope.aliyuncs.com/v1
```

**方案 C: DeepSeek**
```
LLM 后端：DeepSeek
LLM 模型：deepseek-chat
API Key: sk-xxxxxxxxxx
```

---

### 2️⃣ 第二步：添加视频

#### 方式 1: 复制文件到视频目录

```bash
cp /path/to/your/video.mp4 \

```

```bash
ln -s /path/to/your/videos \
      /Users/nannan/.openclaw/workspace/course-ai-helper/data/videos/courses
```

### 3️⃣ 第三步：扫描视频

1. 访问视频库：http://localhost:3000/
2. 点击右上角 **"📁 扫描新视频"** 按钮
3. 系统会自动发现视频文件

---

### 4️⃣ 第四步：处理视频

#### 方法 1: 通过视频列表（推荐）

1. 访问视频库：http://localhost:3000/
2. 找到要处理的视频
3. 点击视频卡片上的 **"🚀 开始处理"** 按钮
4. 确认处理
5. 等待完成

#### 方法 2: 通过视频详情页

1. 点击视频卡片进入详情页
2. 点击 **"开始处理"** 按钮
3. 等待完成

---

## 📊 处理流程

视频处理包含两个步骤：

### 步骤 1: ASR 语音识别
```
视频文件 → ASR 引擎 → 字幕文件 (.srt)
```
- 提取音频
- 语音识别
- 生成字幕
- 时间：约视频时长的 1/10 - 1/2

### 步骤 2: LLM 内容总结
```
字幕文本 → 大模型 → 课程总结 + 知识点
```
- 分析字幕内容
- 生成课程总结
- 提取知识点（带时间戳）
- 时间：约 30 秒 - 2 分钟

## 🎯 处理状态说明

| 状态 | 说明 | 操作 |
|------|------|------|
| **已扫描** | 视频已添加到系统 | 点击"开始处理" |
| **生成字幕中** | ASR 正在处理 | 等待完成 |
| **字幕完成** | 字幕已生成 | 可以观看 |
| **生成总结中** | LLM 正在分析 | 等待完成 |
| **总结完成** | 总结已生成 | 可以查看总结 |
| **处理失败** | 处理出错 | 查看日志，重试 |

## ⚠️ 注意事项

### 1. 处理时间

| 视频时长 | Faster-Whisper | Deepgram |
|----------|---------------|----------|
| 10 分钟 | ~2-3 分钟 | ~30 秒 |
| 30 分钟 | ~5-8 分钟 | ~1 分钟 |
| 60 分钟 | ~10-15 分钟 | ~2 分钟 |

*以上为 ASR 处理时间，不包括总结生成时间*

### 2. 资源占用

**本地 ASR (Faster-Whisper)**:
- CPU: 中等占用
- 内存：1-3GB
- 磁盘：模型 1-3GB

**云端 ASR (Deepgram)**:
- 本地资源占用低
- 需要网络
- 按使用量付费

### 3. 费用参考

**Deepgram ASR**:
- Nova-3: $0.0059/分钟
- 1 小时视频 ≈ $0.35

**阿里百炼 LLM**:
- qwen3.5: ¥0.002/1K tokens
- 1000 字总结 ≈ ¥0.02

## 🐛 故障排查

### Q1: 点击"开始处理"没反应

**解决**:
1. 检查是否已登录
2. 刷新页面重试
3. 查看浏览器控制台（F12）

### Q2: 处理失败

**解决**:
1. 检查 ASR 配置是否正确
2. 检查 LLM 配置是否正确
3. 查看后端日志：
   ```bash
   tail -f logs/backend.log
   ```

### Q3: 处理很慢

**解决**:
1. 使用云端 ASR（Deepgram）
2. 使用更小的 ASR 模型
3. 检查网络状况

### Q4: 总结质量差

**解决**:
1. 使用更大的 LLM 模型
2. 切换到云端 API（阿里百炼/DeepSeek）
3. 检查字幕质量
