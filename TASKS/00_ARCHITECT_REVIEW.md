# Architect Agent 任务：系统架构评审

## 📋 任务说明

请阅读项目文档和代码，全面了解系统架构，然后提供架构评审报告。

## 📚 需要阅读的材料

1. **项目简报**: `AGENT_TEAM_BRIEF.md`
2. **后端代码**: `backend/app/` 目录
   - 重点：`api/`, `models/`, `services/`
3. **数据库设计**: `backend/app/models/` 下的模型文件
4. **API 文档**: 查看 `backend/app/api/` 路由定义

## 🎯 评审重点

1. **系统架构**
   - 当前架构是否合理？
   - 有无明显的架构缺陷？
   - 可扩展性如何？

2. **技术选型**
   - 技术栈选择是否合适？
   - 有无更好的替代方案？
   - 性能瓶颈在哪里？

3. **数据库设计**
   - 表结构设计是否合理？
   - 索引优化是否充分？
   - 有无数据一致性问题？

4. **API 设计**
   - RESTful 规范遵循情况
   - 接口设计是否清晰？
   - 错误处理是否完善？

## 📝 输出要求

请提供一份架构评审报告，包含：
- 当前架构的优点
- 发现的问题和风险
- 改进建议（按优先级排序）
- 技术债务清单

## ⏰ 完成时间

请在阅读完成后输出报告。
